# 技能与规则生成规则

本文件定义设备域 `openspec/skills/device/` 和 `openspec/rules/device/` 的标准化格式、生成流程和质量要求。

新增或修改技能和规则时必须遵循本文件。

---

## 技能文件标准（`openspec/skills/device/`）

### 格式要求

每个技能文件必须包含 **YAML frontmatter + Markdown body**，参照 Cursor SKILL.md 标准。

```markdown
---
name: kebab-case-name
description: >-
  第三人称描述。说明技能做什么、什么时候触发。
---

# 技能标题

简短目标说明。

## 适用场景
## 步骤
## 不做什么
```

### frontmatter 字段

| 字段 | 要求 | 说明 |
| --- | --- | --- |
| `name` | 必填，kebab-case，≤64 字符，仅小写字母/数字/连字符 | 技能唯一标识，用于工具发现和触发 |
| `description` | 必填，≤1024 字符，非空 | 第三人称，包含 WHAT（做什么）和 WHEN（什么时候用） |

### body 结构

| 节 | 必须 | 说明 |
| --- | --- | --- |
| 标题 + 一句话目标 | 是 | 让读者 3 秒内知道这个技能干什么 |
| 适用场景 | 是 | 列出 2-4 个触发条件 |
| 前置条件 | 否 | 如有依赖其他技能或规则，在此声明 |
| 步骤 | 是 | 具体操作，按编号排列 |
| 输出格式 | 否 | 如果技能有固定输出模板，在此给出 |
| 不做什么 | 是 | 明确边界，防止误用 |

### 质量约束

- body 总行数 **< 500 行**
- 假设 agent 已经很聪明，只写它不知道的上下文
- 步骤中的命令/代码块必须可直接执行，不写伪代码
- 引用其他技能/规则时用文件名（如 `04-文档沉淀技能.md`），不用绝对路径

### 命名

- 文件名：`NN-中文名称.md`（NN 为两位编号，按开发生命周期排序）
- `name` 字段：`device-kebab-name`（以 `device-` 前缀标识设备域）

---

## 规则文件标准（`openspec/rules/device/`）

### 格式要求

规则文件使用 **纯 Markdown**，不加 YAML frontmatter（与 Cursor `.mdc` 规则区分）。

```markdown
# 规则标题

一句话说明本规则的目标和适用范围。

## 节 1
## 节 2
```

### 结构

| 节 | 必须 | 说明 |
| --- | --- | --- |
| 标题 + 一句话目标 | 是 | 规则的定位 |
| 适用范围 | 是 | 哪些任务/文件/目录受此规则约束 |
| 具体条款 | 是 | 编号列出，每条可独立引用 |
| 边界/例外 | 否 | 哪些情况不在本规则管辖范围 |

### 命名

- 文件名：`NN-规则名称.md`
- 01-09：目录与元规则（结构、命名、迁移、生成规则等）
- 10-19：专项开发约束（Web Panels、设备上下文等）
- 20+：预留

### 质量约束

- 一个规则文件只管一件事
- 条款可执行、可验证，不写模糊建议
- 需要修改时直接改原文，不另建"补充说明"文件

---

## 生成流程

### 新增技能

1. 确认技能不与现有技能重复 — 先读 `openspec/skills/device/README.md`
2. 确定编号 — 按开发生命周期插入合适位置，如需调整已有编号则批量重命名
3. 按上述格式在 `openspec/skills/device/` 创建原始技能文件
4. 在 `/.cursor/skills/` 创建桥接目录和 `SKILL.md`（见下方"桥接规范"）
5. 更新 `openspec/skills/device/README.md` — 在对应分组中添加条目，包含文件名、`name`、用途
6. 如果新技能被其他技能引用（如 `08-设备开发完成技能.md`），同步更新引用方

### 新增规则

1. 确认规则不与现有规则重复 — 先读 `openspec/rules/device/README.md`
2. 确定编号 — 按编号段含义选择
3. 按上述格式创建文件
4. 更新 `openspec/rules/device/README.md` — 添加文件名和一句话说明

### 修改已有文件

- 小幅修订（补一行、改一个值）→ 直接编辑
- 大幅重构 → 保留原编号和文件名，内容整体替换
- 如需废弃 → 在 README 中标注"已废弃"并说明替代方案，保留文件不删除

## 桥接规范

设备技能的原始内容维护在 `openspec/skills/device/`，通过 `/.cursor/skills/` 下的桥接文件注册到 Cursor 技能系统，
使 Cursor 能自动发现和触发。

### 桥接文件结构

```
/.cursor/skills/<name>/SKILL.md
```

`<name>` 与原始技能的 `name` 字段一致（如 `device-context-entry`）。

### 桥接文件模板

```markdown
---
name: <与原始技能相同>
description: >-
  <与原始技能相同>
---

# <技能标题>（桥接）

仅当 `deviceDevelopProfile` 为 `true` 或用户明确要求进入设备域时生效。

## 门控

1. 读取 `/openspec/ai-context/switch.json` 中 `deviceDevelopProfile`，并检查环境变量 `STUDIO_AI_DEVICE_ENTRY`
2. 如果两路开关均未启用 **且** 用户未明确要求进入设备域 → 跳过，不执行
3. 通过 → 继续

## 执行

读取并遵循 `/openspec/skills/device/<NN-原始文件名>.md` 中的完整指令。
```

### 关键约束

- **桥接文件只做转发**，不包含业务逻辑，所有实际指令在原始文件中维护
- `name` 和 `description` 必须与原始技能文件的 frontmatter 完全一致
- 每次修改原始技能的 `name` 或 `description` 时，必须同步更新桥接文件
- 桥接映射表维护在 `/.cursor/rules/device-doc-context.mdc` 中

---

## 索引一致性

每次新增或修改技能/规则后，必须同步更新对应的 `README.md`。

| 改动位置 | 必须更新 |
| --- | --- |
| `openspec/skills/device/` 新增/修改/重排 | `openspec/skills/device/README.md` |
| `openspec/skills/device/` name/description 变化 | 对应 `/.cursor/skills/<name>/SKILL.md` 桥接文件 |
| `openspec/rules/device/` 新增/修改/重排 | `openspec/rules/device/README.md` |
| 技能间交叉引用变化 | 引用方文件 |
| 桥接映射变化 | `/.cursor/rules/device-doc-context.mdc` |
