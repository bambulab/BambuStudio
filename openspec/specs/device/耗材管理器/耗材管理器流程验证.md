# 耗材管理器流程验证

本文件用于支持耗材管理器（Filament Manager）的人工流程验证，面向开发者记录：

- 日常验证应如何操作
- 每一步应观察什么结果
- 出问题时先查哪条链路
- 当前分支下可用的特殊指令和特殊入口

它不是产品说明书，而是开发者验证手册。

## 1. 适用范围

适用于以下两类实现形态：

### 1. 旧版 WebView 页面

- 前端资源：`resources/web/fila_manager/`
- C++ 面板：`src/slic3r/GUI/fila_manager/wgtFilaManagerPanel.cpp`
- 旧协议特征：`HandleFilaManager`、`{type, seq, command, data}`

### 2. Web Panel 迁移版

- 前端工程：`web-panels/`
- 构建产物：`resources/web/panels/fila-manager/`
- 新协议特征：JSON-RPC 2.0、`fila_manager.xxx`

验证前必须先判断当前分支属于哪一种，否则容易把“预期行为”和“当前实现”混在一起。

## 2. 验证前准备

### 2.1 代码与资源准备

至少确认以下路径之一存在且与你当前分支一致：

- `resources/web/fila_manager/`
- `resources/web/panels/fila-manager/`

如果当前分支是 Web Panel 迁移版，还要确认：

- `web-panels/` 目录存在
- 相关产物已经构建到 `resources/web/panels/fila-manager/`

### 2.2 数据准备

建议准备以下验证数据：

- 至少 2 条已有 spool 数据
- 至少 1 条可编辑的手动录入数据
- 至少 1 条可用于收藏 / 存档 / 删除的测试数据
- 如果验证 AMS 流程，准备已登录且有机器绑定的环境

### 2.3 关联文档

建议同时打开：

- `openspec/specs/device/功能架构文档/filament_manager_spec.md`
- `openspec/specs/filament-manager-panel/spec.md`
- `openspec/specs/filament-ams-sync/spec.md`
- `openspec/changes/migrate-fila-manager-to-web-panels/design.md`

## 3. 标准验证流程

以下流程建议按顺序执行，每一步都记录“操作”和“实际结果”。

### 3.1 页面进入与初始化

#### 操作

1. 启动 Studio
2. 进入耗材管理器页面
3. 观察页面是否完成初始化

#### 预期

- 页面可以正常打开，无白屏
- 基础列表或空状态可以显示
- 当前主题样式正确
- 初始化后可以进行后续交互

#### 重点观察

- 旧版实现：是否收到 `spool_list` / `preset_options` 等初始化数据
- Web Panel 版：是否完成 `fila_manager.init` 或等价初始化响应

### 3.2 列表展示

#### 操作

1. 查看默认列表
2. 核对品牌、类型、颜色、余量、状态等核心字段

#### 预期

- 列表字段完整
- 空状态、非空状态切换正常
- 数据数量与本地存储一致

#### 重点观察

- 是否存在字段错位
- 是否存在列表为空但本地数据不为空
- 是否存在主题切换后样式异常

### 3.3 手动新增

#### 操作

1. 打开“添加耗材”
2. 选择品牌 / 类型 / 系列
3. 选择颜色
4. 输入重量、价格、备注等字段
5. 提交保存

#### 预期

- 表单可完整填写
- 提交后列表新增记录
- 关闭并重新打开页面后记录仍存在

#### 重点观察

- 重量计算是否正确
- 必填项校验是否生效
- 新增后是否立即刷新列表

### 3.4 编辑已有耗材

#### 操作

1. 从列表进入某条 spool 的编辑
2. 修改 1 到 2 个字段
3. 保存并返回列表

#### 预期

- 编辑表单预填正确
- 保存后列表即时更新
- 重启后修改结果仍保留

#### 重点观察

- 编辑模式是否错误地显示批量数量选择器
- 保存后是否出现重复记录而不是更新原记录

### 3.5 收藏、存档、标记空料、删除

#### 操作

1. 对同一条或不同记录分别执行：
   - 收藏 / 取消收藏
   - 存档
   - 标记空料
   - 删除

#### 预期

- 每个操作都能在列表上即时反映
- 收藏标签、存档状态、空料状态正确变化
- 删除后数据真实移除

#### 重点观察

- 状态变化是否只更新 UI、没有落库
- 存档视图与主列表视图是否一致

### 3.6 批量操作

#### 操作

1. 勾选多条记录
2. 执行批量删除或批量添加后的批量校验

#### 预期

- 选中状态正确
- 批量操作结果与选中集一致
- 未选中记录不受影响

#### 重点观察

- 全选 / 反选是否正确
- 批量删除后分页、筛选状态是否异常

### 3.7 筛选、搜索、排序、分页

#### 操作

1. 按品牌 / 类型 / 系列 / 状态筛选
2. 输入关键字搜索
3. 切换排序列
4. 修改分页大小或切换页码

#### 预期

- 结果集符合条件
- 搜索与筛选可以叠加
- 排序方向清晰且结果稳定
- 分页切换不丢筛选状态

#### 重点观察

- 客户端筛选是否与当前列表源数据一致
- 筛选切换后是否错误回到第一页或丢失选择

### 3.8 AMS 模式

#### 操作

1. 进入 AMS 相关入口
2. 获取设备列表
3. 选择设备
4. 获取 AMS 数据
5. 如支持，同步或导入到耗材库

#### 预期

- 有设备时能展示设备列表
- 选机后能拿到对应 AMS 数据
- 无设备时给出可理解提示，不崩溃

#### 重点观察

- 设备列表和 AMS 数据是否解耦
- 切换设备时是否错误复用旧数据
- 同步后列表是否实时更新

### 3.9 重启持久化

#### 操作

1. 完成新增 / 编辑 / 删除后退出 Studio
2. 重新启动并进入耗材管理器

#### 预期

- 数据与退出前一致
- 无丢失、回滚、重复记录

## 4. 常见异常场景

### 4.1 页面空白或初始化失败

先检查：

- 当前分支到底使用 `resources/web/fila_manager/` 还是 `resources/web/panels/fila-manager/`
- 面板加载路径是否与分支实现一致
- 前端资源是否已经构建

### 4.2 操作后 UI 变了但数据没保存

先检查：

- 是否只发生了前端乐观更新
- C++ handler 是否真正执行到 store 写入
- 重启后数据是否回退

### 4.3 AMS 列表拿不到机器

先检查：

- 当前是否登录
- 用户名下是否有机器
- 当前分支是否支持 `get_machine_list` / `fila_manager.get_machine_list`

### 4.4 旧协议与新协议混用

典型症状：

- 前端发的是 `fila_manager.xxx`，C++ 还在等 `command`
- C++ 推的是 `spool_list`，前端却在监听 `fila_manager.spool_list_updated`

出现这种问题时，先统一确认当前分支属于旧版还是 Web Panel 迁移版。
