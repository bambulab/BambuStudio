# 设备域开发套件移植技能

把当前项目的设备域 AI 开发资产打包成 `workspace/` + `globals/` 双层 toolkit，再用对应安装脚本铺到目标项目或新机器。

## 触发方式

- **仅手动调用**：用户明确说"按设备域开发套件移植技能打包" / "把设备域套件移植到 X 项目" / 主动选择 `device-kit-migrate` 技能
- 不跟随 `deviceDevelopProfile` 自动激活
- 触发时**必须先向用户输出告知声明**，例如："正在按设备域开发套件移植技能打包当前项目的设备域资产，会询问 3 个边界问题再执行。"

## 不做什么

- **不**修改源项目的 `.cursor/` / `openspec/` —— 只读打包，不动源
- **不**自动覆盖目标项目同名文件（默认 Merge，必须显式 `-Mode Overwrite` 才覆盖）
- **不**包含 token / 凭据 / `.git/` / 构建产物 / 源码 / `agent-transcripts/` / `terminals/`
- **不**包含第三方插件 cache（`~/.cursor/plugins/`、`~/.claude/plugins/`）
- **不**在未询问用户脱敏意愿前盲目替换 STUDIO 标记
- **不**在用户没明确"开始迁移"前主动跑打包

## 步骤

### 1. 触发告知

向用户输出："正在按设备域开发套件移植技能打包当前项目的设备域资产，会先问 3 个边界问题再执行。"

### 2. 与用户七问

按以下结构化提问，**等用户回答后**再继续。1-5 必问；6-7 必问其一：

| # | 问题 | 选项 / 默认 |
| --- | --- | --- |
| 1 | **目标项目** | 名称（用于命名 toolkit 与脱敏替换；可附路径若选直接铺） |
| 2 | 是否打包**用户名下**（`~/.cursor/` `~/.claude/`）的技能和规则 | 是 / 否（默认是）|
| 3 | 是否打包**项目下其他**（非 device-* 的）通用规则与跨项目技能（B 表）| 是 / 否（默认是）|
| 4 | 是否打包 `openspec/specs/device/`（体量大、绑业务、200+ MD）| 是 / 否（默认否）|
| 5 | 是否对 **STUDIO / 项目专有标记脱敏**（host / JIRA 前缀 / 环境变量名等，详见 §脱敏方案）| 是 / 否（默认是）|
| 6 | 输出形态：(a) 中转 toolkit 目录（路径 `~/Desktop/<target>-dev-toolkit/`） | a（默认）|
| 7 | 输出形态：(b) 直接铺到目标项目根 | 提供 `ProjectPath` 时启用；同时选择 `Merge`（默认）或 `Overwrite` |

### 3. 收集源信息

- 当前项目根：用 `git rev-parse --show-toplevel` 拿
- 当前项目名：取 root 目录名（用于命名 toolkit 与脚本）
- 用户主目录：`$env:USERPROFILE`

### 4. 准备目标目录

输出形态 (a)：

- toolkit 根：`~/Desktop/<target>-dev-toolkit/`（`<target>` 来自问题 1）
- 子目录：`workspace/`、`globals/`（按问题 2-4 决定哪些填、哪些跳）
- 脚本：从 `m2-dev-toolkit/install.ps1` `install-globals.ps1` 复制并替换硬编码字串（toolkit 名、JIRA 前缀提示等）
- README：从 `m2-dev-toolkit/README.md` 复制并替换 M2 / m2-dev-toolkit / `M2_JIRA_*` 等

输出形态 (b)：

- 不生成中转 toolkit，直接调用 `install.ps1 -ProjectPath <user-given> -Mode <Merge|Overwrite>` 风格的拷贝逻辑

### 5. 打包项目级（A + 选中的 B）

按"三层资产盘点"A 表逐项 `Copy-Item -Recurse`：

- 目标：`workspace/<相同相对路径>`
- 排除：`.git/`、`build/`、`out/`、`node_modules/`、`*.lock`、`*.log`
- 编码：保留原文（设备域文档大量中文，使用 UTF-8 / UTF-8-BOM 不要重编码）

按问题 3 / 4 选择追加 B 表项（项目级通用 / `openspec/specs/device/`）。

### 6. 打包用户级（仅当问题 2 = 是）

按 C 表，注意：

- 只打包**自定义**技能：跳过明显是 Cursor 自带或第三方 plugin cache 的目录（`~/.cursor/plugins/cache/**`、`~/.claude/plugins/cache/**`）
- 自定义判定参考：`m2-dev-toolkit/globals/.cursor/skills/` 已列出的目录名是合格白名单
- 如果用户曾用 cursor-public、claude-plugins-official 等插件，目录名带 `cache/` 段都跳

### 7. 脱敏方案（仅当问题 5 = 是）

打包后、生成 README 前，对 toolkit 内所有文本文件按下表替换。**只动 toolkit 副本，不动源仓库**。

| 类别 | 模式（regex） | 默认替换 |
| --- | --- | --- |
| JIRA 项目前缀 | `\bSTUDIO-\d+\b` | `<JIRA-PREFIX>-NNNNN` 或 `${target_jira_prefix}-NNNNN` |
| profile 开关环境变量 | `\bSTUDIO_AI_DEVICE_ENTRY\b` | `<TARGET>_AI_DEVICE_ENTRY` |
| JIRA 凭证环境变量 | `\bSTUDIO_JIRA_USER\b` / `\bSTUDIO_JIRA_TOKEN\b` | `<TARGET>_JIRA_USER` / `<TARGET>_JIRA_TOKEN` |
| Gerrit 主机 | `gerrit\.bambooolab\.com` | `<gerrit-host>` |
| JIRA 主机 | `jira\.bambooolab\.com` | `<jira-host>` |
| Gerrit repo 路径 | `bbl/bamboo_slicer` | `<gerrit-repo-path>` |
| 默认目标分支 | `github/01\.01\.00\.\d+` | `<default-branch>` |
| 项目代号（按用户确认）| `BambuStudio` / `Bambu Lab` 等品牌词 | 不默认替换；仅当用户在问题 1 显式给出替换值时才动 |

**不**脱敏：

- 设备小组成员邮箱与人名（事实记录，不是凭据；目标项目接手后由新 owner 决定是否改）
- C++ 源码中的品牌字串（不属于 toolkit 范围）
- 开源依赖名（Slic3r / wxWidgets / wxPython 等）

执行模板（PowerShell + ripgrep）：

```powershell
$toolkit = '<path-to-toolkit-root>'
$prefix  = '<TARGET>'   # 来自问题 1
$jira    = '<JIRA-PREFIX>'

# 7.1 预览即将命中的文件
rg --hidden -uu '\bSTUDIO[-_]|gerrit\.bambooolab|jira\.bambooolab|bbl/bamboo_slicer' $toolkit

# 7.2 批量替换（仅文本文件）
Get-ChildItem $toolkit -Recurse -File -Include *.md,*.mdc,*.ps1,*.py,*.json,*.html,*.txt |
  ForEach-Object {
    $c = Get-Content $_.FullName -Raw
    $c = $c -replace '\bSTUDIO-(\d+)\b', "$jira-`$1"
    $c = $c -replace '\bSTUDIO_AI_DEVICE_ENTRY\b', "${prefix}_AI_DEVICE_ENTRY"
    $c = $c -replace '\bSTUDIO_JIRA_USER\b',  "${prefix}_JIRA_USER"
    $c = $c -replace '\bSTUDIO_JIRA_TOKEN\b', "${prefix}_JIRA_TOKEN"
    $c = $c -replace 'gerrit\.bambooolab\.com', '<gerrit-host>'
    $c = $c -replace 'jira\.bambooolab\.com', '<jira-host>'
    $c = $c -replace 'bbl/bamboo_slicer', '<gerrit-repo-path>'
    Set-Content $_.FullName -Value $c -Encoding UTF8 -NoNewline:$false
  }

# 7.3 校验（应为空或仅剩接受的残留）
rg --hidden -uu '\bSTUDIO[-_]|gerrit\.bambooolab|jira\.bambooolab|bbl/bamboo_slicer' $toolkit
```

执行完毕在 toolkit README 顶部记录"已脱敏：是 / 否 + 替换映射表"，方便接手者复盘。

### 8. 输出本地化清单（无论是否脱敏都要给）

把以下清单写到生成的 `README.md`，并在控制台 echo 给用户：

- `openspec/ai-context/switch.json` 中 `deviceDevelopProfile` 等开关按目标项目调整
- 检查 `openspec/skills/device/30-Gerrit*` 与 `40-JIRA*` 中 host / repo / branch 占位符是否需要落地
- 检查 `.cursor/skills/jira-*/scripts/*` 凭证环境变量名是否已替换
- 检查 `.cursor/rules/commit-msg.mdc` 的 JIRA 项目前缀
- `openspec/specs/device/`（如带）host 与机型 profile 名占位符落地
- 全仓再 grep 一次：`rg --hidden -uu '<JIRA-PREFIX>|<gerrit-host>|<jira-host>|<gerrit-repo-path>|<default-branch>|<TARGET>'` 替换为目标项目实际值

### 9. 验证

- `ls toolkit/workspace/` 和 `ls toolkit/globals/` 输出顶层
- toolkit 总文件数 / 总体积 echo
- 提示用户："如需安装到新项目：`cd <toolkit>; .\install.ps1 -ProjectPath <NEW_PROJ>`"
- 提示用户："如需安装到当前用户目录：`.\install-globals.ps1`"
