# 设备问题分析技能

把一个问题描述快速拆成"功能语义 + 设计背景 + 源码入口"。

## 适用场景

- 排查设备页面显示异常或状态不一致
- 理解某个设备能力的实现路径
- 需要从问题域追溯到设计决策和代码入口

## 步骤

### 1. 定位问题域

在 `openspec/specs/device/主题索引.md` 中找到问题所属分类。

### 2. 理解功能结构

读 `openspec/specs/device/功能架构文档/README.md`，理解系统分层、页面关系和数据流。

### 3. 追溯代码入口

读 `openspec/specs/device/软件代码文档/README.md`，找到源码入口、协议接口和 OpenSpec 设计背景。

### 4. 进入源码

用 `Grep` / `SemanticSearch` 在源码中验证和深入。

### 5. 网络库外部源码的处理

如果问题链路继续下钻到 `NetworkAgent -> bambu_network -> AccountManager`：

- 先从 `openspec/specs/device/软件代码文档/` 中确认已有归档结论
- 再确认当前工作区是否包含可读的网络库源码
- 若不包含，先让用户提供网络库仓库链接或源码路径
- 拿到链接后再继续做实现级分析，避免把归档结论误当成源码事实

## 输出

分析后应得到三样东西：

- 对应的问题域分类
- 对应的资料入口（文档路径）
- 对应的源码入口或变更路径

## 不做什么

- 不跳过文档直接猜源码位置
- 不在没理解功能结构的情况下修改代码
- 不把分析过程中的临时猜测当作结论
- 不在缺少外部网络库源码时，直接推断 `bambu_network` 内部实现细节
