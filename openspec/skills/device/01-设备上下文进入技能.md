# 设备上下文进入技能

在设备研发任务开始时，快速而准确地进入 `/openspec/docs/device/`，避免全盘扫描或漏掉关键规则。

## 适用场景

- 开始一个设备域开发或排查任务
- 不确定当前任务是否属于设备域
- 需要加载设备相关规则和知识

## 步骤

### 1. 判断是否启用设备上下文

读取 `/.ai-context/switch.json`，仅当 `deviceDevelopProfile` 为 `true` 且任务属于设备域时继续。

### 2. 标准进入顺序

按顺序进入，不要跳步：

1. `/openspec/docs/device/readme食用指南.md`
2. `/openspec/docs/device/开发流程.md`
3. 若涉及 OpenSpec → `/openspec/rules/device/03-openspec规则.md`
4. `/openspec/specs/device/README.md`
5. 需要快速定位问题域 → `/openspec/specs/device/主题索引.md`

### 3. 按任务类型分流

| 任务类型 | 入口 |
| --- | --- |
| 目录调整 / 资料迁移 / 归档整理 | `openspec/rules/device/README.md` |
| OpenSpec 变更 / 规格 / 归档 | `openspec/rules/device/03-openspec规则.md` → `openspec/specs/device/` |
| 问题分析 / 资料检索 / 经验沉淀 | `openspec/skills/device/README.md` |
| 功能 / 代码入口 / 历史方案 | `openspec/specs/device/主题索引.md` + `openspec/specs/device/来源映射表.md` |

## 不做什么

- 不机械读取整个 `/openspec/docs/device/`
- 不在设备 profile 关闭时强行加载设备上下文
- 入口→索引→专题的顺序优先于全量搜索
