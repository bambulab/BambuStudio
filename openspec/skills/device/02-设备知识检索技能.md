# 设备知识检索技能

在最短时间内找到当前设备问题应该先看哪份资料。

## 适用场景

- 不知道某个设备功能的文档在哪
- 需要从问题域反查设计背景或源码入口
- 需要确认某个设备能力是否已有文档记录

## 步骤

### 1. 索引优先

1. 先看 `openspec/specs/device/主题索引.md` — 按问题域定位
2. 再看 `openspec/specs/device/来源映射表.md` — 反查原始路径
3. 最后才进入原始路径或源码搜索

### 2. 使用工具

| 工具 | 用途 |
| --- | --- |
| `Glob` | 找目录结构和文件 |
| `Read` | 读入口页和专题文档 |
| `Grep` | 搜术语、类名、协议名、函数名 |
| `SemanticSearch` | 追某个行为是怎么工作的 |

### 3. 常用检索入口

| 问题域 | 主题索引中的节 |
| --- | --- |
| 设备对象 / 设备列表 | 设备对象与列表管理 |
| 设备状态同步 | 设备状态同步与订阅 |
| 发送打印 | 发送打印与选机流程 |
| AMS / 耗材 | AMS、料槽与耗材同步 |
| 耗材管理器 | 耗材管理器 |

### 4. 网络库相关源码的检索约束

当问题涉及 `NetworkAgent`、`bambu_network`、`AccountManager` 或“网络库导出接口”时：

1. 先看 `openspec/specs/device/主题索引.md` 的“发送协议、任务与回执”与“耗材管理器”入口
2. 再看 `openspec/specs/device/来源映射表.md` 中 networking / Studio 适配实现的映射
3. 如果当前工作区没有可读的网络库源码，或路径已失效，不要猜实现细节
4. 先向用户索要网络库仓库链接或源码位置，再继续进入实现层分析

## 不做什么

- 不一开始就只搜源码，先建立资料入口
- 不跳过索引直接全仓库 grep
- 如果资料已在 `openspec/specs/device/` 中有入口，优先从归档入口进入
- 涉及外部网络库时，不在没有源码的情况下臆测 `AccountManager` / `bambu_network` 内部实现
